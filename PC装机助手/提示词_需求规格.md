# PC 装机助手 — 需求规格与开发提示词

> 版本 v1.0 | 2025-07-13
> 使用方式：将本文档全文复制给 AI（Claude/DeepSeek/GPT 等），作为开发规格书。

---

## 一、项目概述

构建一个**单页 Web 应用**（HTML+CSS+JS，单个文件可运行），功能是：

1. **爬取/展示**中国主流电商平台（京东、淘宝、拼多多、抖音）的电脑配件价格
2. **引导用户**在左侧边栏逐项选择配件，自动完成兼容性过滤
3. **右侧展示**配置清单、总价、以及游戏性能 + AI 性能评估

目标用户：想自己装机但不确定配件是否兼容、性能是否够用的消费者。

---

## 二、技术约束

| 项目 | 要求 |
|------|------|
| 交付物 | 单个 `pc-builder.html` 文件（CSS/JS 全部内嵌） |
| 打开方式 | 浏览器直接双击即可使用 |
| UI 风格 | 暗色主题（深灰/黑底 + 蓝/绿强调色），科技感/极客风 |
| 响应式 | 桌面端优先（≥1366px 宽度），移动端可简化为单列布局 |
| 第三方依赖 | **零依赖**，不引入任何 CDN/框架 |
| 数据来源 | 初始使用**内置静态数据**（2025 年中市场行情），预留爬虫接口位 |

> **为什么初始用静态数据而非实时爬取**：跨域限制下浏览器端无法直接爬淘宝/京东等页面。架构上应设计为：默认加载内置 JSON 数据 → 预留一个「更新价格」按钮 → 按钮实际调用后端 API（另行开发）。本文档侧重前端交互逻辑。

### 3.2 交互行为

1. **折叠/展开**：每个板块可点击标题折叠，默认全部展开。
2. **品牌切换**：CPU 板块选 AMD → 型号列表只显示 AMD CPU；GPU 同理。
3. **价位过滤**：四个价位对应大致范围——
   - 入门 < ¥800
   - 中端 ¥800–¥2000
   - 高端 ¥2000–¥4000
   - 旗舰 > ¥4000
4. **型号卡片**：每项至少显示 `名称 + 核心规格 + 价格`；GPU 额外显示 `显存大小 + 显存位宽 + 显存类型`。
5. **已选标记**：选中的配件以高亮边框 + ✓ 图标标记。
6. **悬停提示**：鼠标悬停显示详细规格 tooltip。

## 四、核心逻辑 — 兼容性引擎

### 4.1 CPU → 主板 兼容规则

| CPU 插槽 | 兼容主板芯片组 | 备注 |
|----------|---------------|------|
| LGA1700 (Intel 12/13/14 代) | H610, B660, B760, Z690, Z790 | H610 供电弱，不支持 i7/i9 |
| AM5 (AMD Ryzen 7000/9000) | A620, B650, B650E, X670, X670E | A620 供电弱，不支持 R9 |
| AM4 (AMD Ryzen 5000) | B450, B550, X570 | 老旧平台，仅入门推荐 |

**供电过滤逻辑**：
- 低功耗 CPU（TDP ≤ 65W）：所有主板可用
- 中功耗 CPU（TDP 65W–125W）：过滤掉 H610 / A620 等丐版
- 高功耗 CPU（TDP > 125W 或 i9/R9）：仅显示 B760 中高端 / Z790 / B650E / X670 及以上

### 4.2 主板 → 内存 兼容规则

- 主板明确标注 DDR4 → 内存列表仅显示 DDR4
- 主板明确标注 DDR5 → 内存列表仅显示 DDR5
- 部分主板同时支持 DDR4/DDR5（极少）→ 两种都显示，但提示"不可混插"

### 4.3 CPU + GPU → 电源 推荐规则

```
推荐瓦数 = (CPU_TDP + GPU_TDP) × 1.5 + 100
```
- 向上取整到 50W 的倍数（550W, 600W, 650W…）
- 电源列表仅显示 ≥ 推荐瓦数的型号
- UI 提示：「根据你的配置，建议至少 XXX W 电源」

### 4.4 机箱 → 主板/显卡 兼容

- 机箱标注支持的主板板型（ATX/mATX/ITX）
- 机箱标注显卡限长 → GPU 长度 > 限长则提示不兼容
- 机箱标注散热器限高 → 风冷高度 > 限高则提示不兼容

### 4.5 运行时行为

- **上游选择变化 → 下游自动清空并过滤**：CPU 变了 → 清空主板/内存选择，重新过滤可选列表
- **不兼容的选项灰色显示**而非隐藏，悬停提示原因（如"需要 LGA1700 插槽"）
- 顶部总览栏实时显示已选配件数量 及 兼容性状态（✅ 全部兼容 / ⚠️ X 项不兼容 / ❌ 缺少必选项）

```
│ │ Llama 3 70B      │ ❌ 显存不足   │          │

│ • 约 85–110 tokens/秒                         │
│ • Prompt 处理: ~2500 tokens/秒                │
│                                              │
│ 💡 推荐部署方案: Ollama + Open WebUI           │
│ 12GB 显存可流畅运行 13B 以下模型，             │
│ 支持 8K context window                        │
└─────────────────────────────────────────────┘
```

#### AI 估算逻辑

| 显存 | 可部署模型（Q4 量化） | 不可部署 |
|------|---------------------|---------|
| 8GB | 7B–8B 模型 | 13B 及以上 |
| 12GB | 7B–14B 模型 | 20B 及以上 |
| 16GB | 7B–20B 模型 | 30B 及以上 |
| 20GB | 7B–30B 模型 | 40B 及以上 |
| 24GB | 7B–34B 模型、70B (IQ3) | 70B Q4 及以上 |

**Token 生成速度** ≈ `显存带宽(GB/s) ÷ 模型单 Token 计算量(GB/token)`

- 7B Q4 模型 ≈ 4.5 GB 参数 + KV Cache ≈ 2 GB → 约 5.5–6.5 GB 显存占用
- 单 Token ≈ 模型参数量的 2× 数据搬运 → 7B Q4 ≈ 9 GB / Token
- 速度 ≈ 带宽 ÷ 9 → RTX 4090 (1008 GB/s) ≈ 112 tok/s；RTX 4060 (272 GB/s) ≈ 30 tok/s

## 九、实现步骤建议

1. **Phase 1**: HTML 骨架 + CSS 暗色主题 + 左侧边栏静态布局
2. **Phase 2**: 硬件数据库（JS 对象，覆盖 CPU/GPU/主板/内存/硬盘/电源/机箱/散热）
3. **Phase 3**: 交互逻辑——品牌切换、价位过滤、型号选择、折叠面板
4. **Phase 4**: 兼容性引擎——CPU→主板、主板→内存、功耗→电源推荐
5. **Phase 5**: 性能评估——游戏基准帧率矩阵 + AI 推理能力计算
6. **Phase 6**: 总价汇总 + 配置清单导出 + 细调 UI 动效
7. **Phase 7**: 兼容性检查 + 边界情况处理 + 响应式适配
